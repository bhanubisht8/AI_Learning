Save
CLI Equivalent
config user local

If you're using AD authentication via RADIUS:

Configure Framed-IP-Address on the RADIUS server for that user
FortiGate will assign that IP on VPN connection

Useful when many users need fixed IPs.

Only Tunnel Mode supports static IP assignment
Avoid overlapping IPs with DHCP/server networks

Short answer: Yes — you can assign a static IP to an SSL-VPN user using Windows Server 2019 (NPS/RADIUS) by returning the Framed-IP-Address RADIUS attribute from the RADIUS server. FortiGate supports that attribute and will use the IP returned by the RADIUS Access-Accept (provided the FortiGate is set to accept RADIUS IP assignment).

Below I give you the practical options and steps you can try right away (no more questions from me — I’ll assume you want server-side assignment from NPS on Windows Server 2019).

Option A — Configure NPS to return Framed-IP-Address (recommended)

Use this when you want the RADIUS server itself to send a specific IP back in the Access-Accept:

Open Server Manager → Tools → Network Policy Server (NPS) on your Windows Server 2019.
Go to Policies → Network Policies and create a policy (or edit an existing policy) that matches the user or group you want to give a static IP to.
In the policy Properties click the Settings tab.
Under RADIUS Attributes → Standard, click Add.
Select Framed-IP-Address and enter the IPv4 address you want the user to get (e.g. 192.168.10.14). Save.

Notes:

This sends the Framed-IP-Address in the Access-Accept for that policy match (so you can use multiple policies to return different IPs to different users/groups).
If you need per-user unique IPs and don’t want many policies, you can also set static IPs on AD user objects (see Option B), or script policy creation.
Option B — Use Active Directory “Dial-in / Assign Static IP” on the user

Windows AD user objects have a Dial-In option where you can set a static IP. In many deployments this is used with RADIUS (NPS) so the AD static IP can be returned as Framed-IP-Address. Steps:

In Active Directory Users and Computers, open a user → Dial-In tab → Assign Static IP addresses → add the IPv4.
Ensure NPS/your Network Policy is set to authorize and return attributes; test login.

Caveat: Some admins report that NPS doesn’t automatically translate the AD Dial-In static IP into a Framed-IP-Address in every scenario — if you see that happening, fall back to Option A (explicit Framed-IP in the NPS policy) which is deterministic.

FortiGate side — make sure FortiGate will accept the RADIUS IP
FortiGate does support the Framed-IP-Address attribute and can use the IP returned by RADIUS for SSL-VPN users. (Fortinet docs & tech-notes show this).
Some FortiGate setups require enabling the behavior in config (or simply proper RADIUS server config). If you find FortiGate still assigns an IP from the SSL-VPN pool instead of the RADIUS IP, search the FortiGate docs/CLI for enabling RADIUS IP override (many community posts mention a small CLI/config tweak). Example resources showing the workflow are Fortinet Community technical note and forum threads.
Troubleshooting checklist (if it doesn’t work)
Verify NPS actually sends Framed-IP-Address (capture RADIUS packets with Wireshark on the server or check NPS logs). If the attribute is missing, FortiGate won’t receive it.
Ensure the IP you return does not conflict with other subnets or DHCP pools.
Confirm FortiGate is configured to accept RADIUS attributes from that RADIUS server and that the RADIUS shared secret and NAS-IP are correct.
If you used AD Dial-In and it didn’t work, try adding Framed-IP directly in the NPS network policy (Option A) — that usually fixes it.
TL;DR
Yes, Windows Server 2019 (NPS) can return Framed-IP-Address to assign a static IP to SSL-VPN users. Configure it in an NPS Network Policy (Settings → RADIUS Attributes → Framed-IP-Address) or set the static IP in the AD user Dial-In properties (but Option A is the most reliable). FortiGate supports using that attribute — just ensure the FortiGate/RADIUS configuration allows RADIUS IP assignment.
