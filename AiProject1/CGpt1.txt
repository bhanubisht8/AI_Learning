👉 LDAP will be used ONLY for password change, not for VPN authentication.

So:

✔ RADIUS → handles login + static IP
✔ LDAP → allows password update in AD
⚠️ Mandatory Requirement (VERY IMPORTANT)

👉 Password change via LDAP works ONLY with LDAPS (secure LDAP)

LDAP (389) ❌ Not allowed
LDAPS (636) ✔ Required
🧭 STEP 1 — Enable LDAPS on AD Server
🔹 1. Install Certificate on Domain Controller

You need a certificate for:

Server Authentication
FQDN = your AD server name
Quick Method:
Open:

Certificates → Computer Account

Personal → Certificates

✔ After install:

Restart service:

net stop ntds
net start ntds

OR reboot server

🔹 2. Verify LDAPS

Run on AD server:

ldp.exe
Connect → Port 636
Enable SSL

✔ If connects → LDAPS working

🧭 STEP 2 — Configure LDAP on FortiGate
🔹 CLI Configuration

Run:

config user ldap
edit "AD-LDAP"
    set server "10.10.10.5"
    set cnid "sAMAccountName"
    set dn "dc=cctnsad,dc=local"
    set type regular
    set username "CN=Administrator,CN=Users,DC=cctnsad,DC=local"
    set password <your_password>
    set secure ldaps
    set port 636
end
🔹 Key Parameters Explained
Field	Meaning
server	AD server IP
cnid	login attribute
dn	base DN
username	bind account
secure ldaps	enable SSL
port 636	LDAPS
